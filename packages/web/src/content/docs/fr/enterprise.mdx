LocalCode Enterprise est destiné aux organisations qui souhaitent s'assurer que leur code et leurs données ne quittent jamais leur infrastructure. Cela est possible grâce à une configuration centralisée qui s'intègre à votre SSO et à votre passerelle IA interne.

:::note
LocalCode ne stocke aucune de vos données de code ou de contexte.
:::

Pour démarrer avec LocalCode Enterprise :

1. Faites un essai en interne avec votre équipe.
2. **<a href={email}>Contactez-nous</a>** pour discuter des options de tarification et de mise en œuvre.

## Essai

LocalCode est open source et ne stocke aucune de vos données de code ou de contexte, vos développeurs peuvent donc simplement [commencer ](/docs/) et effectuer un essai.

---

### Traitement des données

**LocalCode ne stocke pas votre code ni vos données contextuelles.** Tous les traitements s'effectuent localement ou via des appels API directs à votre fournisseur d'IA.

Cela signifie que tant que vous faites appel à un fournisseur de confiance ou à une passerelle IA, vous pouvez utiliser LocalCode en toute sécurité.

La seule mise en garde ici concerne la fonctionnalité facultative `/share`.

---

#### Partager des conversations

Si un utilisateur active la fonctionnalité `/share`, la conversation et les données qui y sont associées sont envoyées au service que nous utilisons pour héberger ces pages de partage sur localcode.dev.

Les données sont actuellement servies via le réseau périphérique de notre CDN et sont mises en cache en périphérie, à proximité de vos utilisateurs.

Nous vous recommandons de désactiver cette option pour votre essai.

```json title="localcode.json"
{
  "$schema": "https://localcode.dev/config.json",
  "share": "disabled"
}
```

[En savoir plus sur le partage](/docs/share).

---

### Configuration centrale

Nous pouvons configurer LocalCode pour utiliser une seule configuration centrale pour l'ensemble de votre organisation.

Cette configuration centralisée peut s'intégrer à votre fournisseur SSO et garantit que tous les utilisateurs accèdent uniquement à votre passerelle IA interne.

---

### Intégration SSO

Grâce à la configuration centrale, LocalCode peut s'intégrer au fournisseur SSO de votre organisation pour l'authentification.

Cela permet à LocalCode d'obtenir les informations d'identification de votre passerelle IA interne via votre système de gestion des identités existant.

---

### Passerelle IA interne

Avec la configuration centrale, LocalCode peut également être configuré pour utiliser uniquement votre passerelle IA interne.

Vous pouvez également désactiver tous les autres fournisseurs d'IA, en vous assurant que toutes les demandes transitent par l'infrastructure approuvée de votre organisation.

---

### Auto-hébergement

Bien que nous vous recommandons de désactiver les pages de partage pour garantir que vos données ne quittent jamais votre organisation, nous pouvons également vous aider à les auto-héberger sur votre infrastructure.

Ceci est actuellement sur notre feuille de route. Si vous êtes intéressé, **<a href={email}>faites-le-nous savoir</a>**.

<details>
<summary>Pouvons-nous utiliser notre propre registre privé NPM ?</summary>

LocalCode prend en charge les registres npm privés via la prise en charge native des fichiers `.npmrc` de Bun. Si votre organisation utilise un registre privé, tel que JFrog Artifactory, Nexus ou similaire, assurez-vous que les développeurs sont authentifiés avant d'exécuter LocalCode.

Pour configurer l'authentification avec votre registre privé :

```bash
npm login --registry=https://your-company.jfrog.io/api/npm/npm-virtual/
```

Cela crée `~/.npmrc` avec les détails d'authentification. LocalCode le détectera automatiquement.

:::caution
Vous devez être connecté au registre privé avant d'exécuter LocalCode.
:::

Alternativement, vous pouvez configurer manuellement un fichier `.npmrc` :

```bash title="~/.npmrc"
registry=https://your-company.jfrog.io/api/npm/npm-virtual/
//your-company.jfrog.io/api/npm/npm-virtual/:_authToken=${NPM_AUTH_TOKEN}
```

Les développeurs doivent être connectés au registre privé avant d'exécuter LocalCode pour garantir que les packages peuvent être installés à partir du registre de votre entreprise.

</details>
